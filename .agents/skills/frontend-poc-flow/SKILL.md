# Frontend PoC Flow

Use this skill for frontend work in `frontend/`.

## Focus

- Keep the UI optimized for local demo clarity.
- Support the three PoC roles: `DONOR`, `CHARITY_MANAGER`, `ADMIN`.
- Show balances, histories, and timeline state clearly.
- Favor direct, readable flows over framework-heavy patterns.

## Working Rules

- Keep one React app with TypeScript and Vite.
- Use mock or local API data until backend endpoints are ready.
- Make the main demo path easy to click through without hidden setup.
- Preserve existing visual language unless the task asks for redesign.

## Expected Outputs

- Pages or sections for the main donation-point flow
- Clean routing and API access patterns
- Role-aware UI states
- Basic loading, empty, and error handling where needed

## Verify

- Install dependencies if needed.
- Run the frontend build after significant changes.
